# Ralph Progress Log

## Codebase Patterns

- `frontend/game.js:applySceneStateEvent()` should always pair guest-office reconciliation with an explicit owner payload update; when a snapshot has no `isMain === true` agent, send the Star UI through the shared idle fallback instead of leaving the previous owner state latched.
- `frontend/game.js:getMainAgentPayloadFromStateSnapshot()` should only derive Star ownership from an agent payload with `isMain === true`; snapshot guests staying online must not become the owner fallback.
- `frontend/game.js:normalizeBackendAgentPayload()` should own alias-vs-name precedence for backend agent labels; prefer a non-empty `alias` before `name` there so snapshot and SSE agent rendering stay aligned.
- `frontend/game.js:normalizeBackendAgentPayload()` should also own the shared presence fallback: when backend payloads omit `authStatus` but include boolean `online`, derive `approved` for `true` and `offline` for `false` there so snapshot and SSE styling stay consistent.
- `frontend/game.js` should normalize compat `/agents` response entries through `normalizeBackendAgentPayload()` before slot assignment and `reconcileOfficeAgentsFromPayload()`, otherwise polling can drift from snapshot/SSE label and presence styling.
- `frontend/game.js` should normalize backend agent-shaped payloads through `normalizeBackendAgentPayload(rawAgent, options)` and layer snapshot-specific field sources such as alias, online-derived presence, and `lastSeenAt` timestamps through the helper's option callbacks instead of duplicating object mapping logic.
- UI routes receive `getPublicState()` snapshots; any mutation that must persist or affect derived room membership should go through `src/state.ts` helpers, especially `updateAgentState()` and `rebuildRooms()`.
- `getPublicState()` must deep-copy nested collections (`agents`, `rooms`, memos, config/background objects) before returning snapshots; shallow copies expose live state to REST/SSE callers.
- The test suite imports built files from `dist/`; keep `npm test` responsible for rebuilding first so contributors and CI always run tests against fresh output.
- The AWN SDK calls the same `onLeave(agentId)` hook for explicit `world.leave` and idle eviction; distinguish the two inside `src/hooks.ts` by comparing the agent's `lastSeenAt` to `manifest.lifecycle.idleTimeoutMs`.
- Presence SSE names should mirror the inferred leave path in `src/hooks.ts`: idle eviction broadcasts `agent_offline`, while voluntary exits broadcast `agent_leave`.
- Hook actions should treat `MemoStore` persistence as best-effort after canonical in-memory updates so transient filesystem failures do not break runtime action success.
- `MemoStore` startup and history loading should treat directory-creation failures and unreadable JSON as recoverable; when scanning prior days, skip corrupt files and keep looking for the most recent valid memo payload.
- Manifest `state_fields` should be sourced from the same canonical field list as `getPublicState()` instead of being duplicated in hook metadata.
- Browser-facing SSE regressions can use an `SSEManager` client double with a minimal `reply.raw` EventEmitter to assert exact streamed event payloads without booting Fastify.
- `frontend/game.js` currently centralizes sync entry points in `create()` and `update()`: `fetchStatus()` owns main-agent UI/status mutation, while `fetchAgents()` owns remote office-agent reconciliation, so SSE work should reuse those boundaries and gate polling at the update loop.
- `frontend/game.js` should route main-agent state/detail updates through `applyMainAgentPayload(payload)` so fetch bootstrap, compat refreshes, and upcoming SSE state events share the same animation/UI mutation path.
- `frontend/game.js` should route remote office-agent payloads through `applyOfficeAgentPayload(agent)`, which delegates to shared `createOfficeAgent()`, `updateOfficeAgent()`, `removeOfficeAgent()`, and `markOfficeAgentOffline()` helpers so polling and future SSE events do not duplicate sprite lifecycle logic.
- `frontend/game.js` should keep SSE transport health on the scene-owned `sseSync` object and funnel raw `/ui/events` payloads through `parseSSEEventPayload(eventType, rawData)` so malformed JSON and partial agent events degrade safely to `null`.
- `frontend/game.js` should open `/ui/events` only through `connectOfficeEventStream(scene)` during `create()`, with `registerOfficeEventStream()` owning the `open`/`error` health transitions so later SSE state and agent handlers reuse the same lifecycle-managed EventSource.
- `frontend/game.js` should treat the `/ui/events` `state` event as a full `getPublicState()` snapshot: derive main-agent status only from an explicit `isMain === true` agent or fall back to the shared idle owner payload, then translate `state.agents` back through `reconcileOfficeAgentsFromPayload()` instead of assuming the `/status` or `/agents` response shapes.
- `frontend/game.js` should route incremental `agent_join`/`agent_update`/`agent_leave`/`agent_offline` events through `applyOfficeAgentStreamEvent()` plus `assignOfficeAgentSlot()` so live SSE updates preserve existing area slots, reuse the shared office-agent lifecycle helpers, and avoid creating ghost offline agents.
- `frontend/game.js` should gate recurring `/status` and `/agents` polling through `shouldUsePollingSync(scene)`, which reads `scene.sseSync.usePollingFallback` so healthy SSE suppresses duplicate fetches and degraded SSE restores both polling loops from one switch.
- `frontend/game.js` should treat `/ui/events` teardown as a first-class scene lifecycle concern: bind one shutdown/destroy cleanup path that removes stored EventSource listeners, closes the active stream, clears reconnect timers, and only creates a fresh EventSource after `readyState === CLOSED`.
- For the current SSE ownership bug surface, `frontend/game.js:getOfficeAgentsFromStateSnapshot()` is the initial snapshot normalization path, `parseSSEAgentPayload()` is the incremental `agent_join`/`agent_update` normalization path, `getMainAgentPayloadFromStateSnapshot()` is the snapshot main-agent selector, and `src/ui-routes.ts:/status` owns the idle fallback behavior.
