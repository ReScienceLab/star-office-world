# Ralph Progress Log

## Codebase Patterns

- UI routes receive `getPublicState()` snapshots; any mutation that must persist or affect derived room membership should go through `src/state.ts` helpers, especially `updateAgentState()` and `rebuildRooms()`.
- `getPublicState()` must deep-copy nested collections (`agents`, `rooms`, memos, config/background objects) before returning snapshots; shallow copies expose live state to REST/SSE callers.
- The test suite imports built files from `dist/`; keep `npm test` responsible for rebuilding first so contributors and CI always run tests against fresh output.
- The AWN SDK calls the same `onLeave(agentId)` hook for explicit `world.leave` and idle eviction; distinguish the two inside `src/hooks.ts` by comparing the agent's `lastSeenAt` to `manifest.lifecycle.idleTimeoutMs`.
- Presence SSE names should mirror the inferred leave path in `src/hooks.ts`: idle eviction broadcasts `agent_offline`, while voluntary exits broadcast `agent_leave`.
- Hook actions should treat `MemoStore` persistence as best-effort after canonical in-memory updates so transient filesystem failures do not break runtime action success.
- `MemoStore` startup and history loading should treat directory-creation failures and unreadable JSON as recoverable; when scanning prior days, skip corrupt files and keep looking for the most recent valid memo payload.
- Manifest `state_fields` should be sourced from the same canonical field list as `getPublicState()` instead of being duplicated in hook metadata.
- Browser-facing SSE regressions can use an `SSEManager` client double with a minimal `reply.raw` EventEmitter to assert exact streamed event payloads without booting Fastify.
- `frontend/game.js` currently centralizes sync entry points in `create()` and `update()`: `fetchStatus()` owns main-agent UI/status mutation, while `fetchAgents()` owns remote office-agent reconciliation, so SSE work should reuse those boundaries and gate polling at the update loop.
- `frontend/game.js` should route main-agent state/detail updates through `applyMainAgentPayload(payload)` so fetch bootstrap, compat refreshes, and upcoming SSE state events share the same animation/UI mutation path.
- `frontend/game.js` should route remote office-agent payloads through `applyOfficeAgentPayload(agent)`, which delegates to shared `createOfficeAgent()`, `updateOfficeAgent()`, `removeOfficeAgent()`, and `markOfficeAgentOffline()` helpers so polling and future SSE events do not duplicate sprite lifecycle logic.
- `frontend/game.js` should keep SSE transport health on the scene-owned `sseSync` object and funnel raw `/ui/events` payloads through `parseSSEEventPayload(eventType, rawData)` so malformed JSON and partial agent events degrade safely to `null`.
